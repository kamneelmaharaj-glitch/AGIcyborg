# 🤝 Contributing — AGIcyborg

Thank you for your intention to serve this mission.

## Ground Rules
- Act in alignment with **Dharma**: clarity, compassion, integrity.
- No secrets in issues/PRs. Never attach `.env`, keys, or runtime binaries.
- Keep changes **small, focused, and tested**.

## Dev Setup
- Follow `DEVELOPMENT.md`.
- Run `python -m tools.validate_env` before opening a PR.
- Include a brief rationale in your PR description.

## Code Style
- Python: readable, typed where helpful, small functions.
- Security-sensitive code: comment **intent** as well as **mechanism**.
- Prefer explicitness over cleverness.

## Licensing
- Runtime is encrypted + licensed. Do not attempt to bypass the loader or protections.
